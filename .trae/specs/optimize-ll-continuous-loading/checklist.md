# Checklist

- [x] LLA/LLB 的上料互锁不再以另一侧整组 wafer 全部回 LP为唯一解锁条件
- [x] LL 下层一旦存在已完成工艺 wafer，系统优先执行下料回 LP，该优先级高于新的 LL 上料
- [x] 当目标 LL 上层有空槽且不存在真实下料冲突时，系统允许继续上料
- [x] 当仍有在制 wafer 依赖 EFEM 下料锁时，已完成返回的 task 不会被提前从 EFEM_RETURN/COMPLETED 重置为 UNKNOWN/UNKNOWN
- [x] 循环完成与重跑仅在安全点触发，不会让仍在途 wafer 因状态丢失进入错误分支
- [ ] 连续跑 25 片场景下，PM 可保持持续供片，不因整组先回 LP 再开下一组造成不必要等待
- [ ] 连续跑片时 WTR 尽量维持交互片节奏，避免“上层持续塞满待加工片、下层完成片迟迟不回 LP”导致堵塞
- [ ] 当 `case 1051` 发现一手持工艺完成片、一手为空且 LL 仍有待加工片时，系统会先把工艺完成片放回 LL，并在 `case 2065` 后续关门/破真空收尾前立即补取一片，为下一次 PM 交互片做准备
- [ ] `executeTMTransfer()` 能正确配合 LL 发起的交互准备，不重复发起冲突动作，也不破坏最小改动入口
- [ ] 日志或诊断中不存在新增的异常状态迁移或明显死锁/卡死迹象

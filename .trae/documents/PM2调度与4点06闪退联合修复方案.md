### 2. 把共享任务数组改为“线程内局部快照”，消除跨线程并发读写同一 `std::vector` 成员
- 文件：[slot_transfer_cycle_vtm_widget.cpp](file:///d:/HLPrj/HL/device/src/slot_transfer_cycle_vtm_widget.cpp)
- 位置：`UpdateLLASubTransferDatas / UpdateLLBSubTransferDatas / UpdateEfemSubTransferDatas / UpdatePmSubTransferDatas` 及所有依赖这些成员数组的线程逻辑
- 修改内容：
  - 不再让多个线程长期读写 `Private` 对象上的共享 `std::vector<UnifiedWaferTask>` 成员作为“最新快照”。
  - 改为按线程/按步骤读取 `TaskManager` 后保存到局部变量或线程私有快照结构，再基于该局部快照完成整段判定与更新。
  - 第一批必须优先收口的路径：
    - `executeLLATransfer`
    - `executeLLBTransfer`
    - `executeEFEMTransfer`
    - `executePM1Transfer`
    - `executePM2Transfer`
    - `executePM3Transfer`
    - `executePM4Transfer`
  - 尤其收口以下高风险共享成员：
    - `loadLockAPendingTasks / loadLockAReturnPendingTasks / loadLockACompletedTasks`
    - `loadLockBPendingTasks / loadLockBReturnPendingTasks / loadLockBCompletedTasks`
    - `efemPendingTasks / efemReturnPendingTasks / efemReturnCompletedTasks`
    - `pmPendingTasks / pmCompletedTasks / pm2PendingTasks / pm2CompletedTasks / pm3PendingTasks / pm4PendingTasks`
- 为什么这样改：
  - 这是当前最像 `0xc0000374` 根因的实现缺陷。
  - 只要继续让多个线程并发覆盖同一批 `std::vector` 成员，调度逻辑即使语义正确，也仍可能随机写坏堆。
